# PokeFireRed Feature Engineering

## Overview

Implement gameplay features by choosing the right subsystem first, editing source-of-truth files only, and validating with focused tests before broad checks. Keep technical implementation in the main agent.

## Workflow

1. Classify the request:
- Combat/move/ability/status logic.
- Item or bag behavior.
- Shop or mart behavior.
- Script-facing integration (new script command, `special`, or script flow hook).

2. Load references by need:
- Always read [workflow-and-file-ownership.md](references/workflow-and-file-ownership.md) first.
- For battle mechanics, read [combat-systems.md](references/combat-systems.md).
- For items/shops, read [items-and-shops.md](references/items-and-shops.md).
- For player-facing copy/theme support, read [design-consultant.md](references/design-consultant.md).
- Before finalizing, follow [validation-and-testing.md](references/validation-and-testing.md).

3. Decide scope:
- Prefer data/config edits when behavior is already supported by engine tables.
- Add C logic only when data/config cannot express the feature.
- Add new script opcodes only when `special`/existing commands cannot cover the use case.

4. Implement with source-of-truth discipline:
- Do not hand-edit generated map includes, autogenerated trainer headers, or generated constants.
- Keep file changes minimal and subsystem-local.

5. Validate:
- Run `make -j8`.
- Run targeted tests for touched mechanics, then `make check` for broad behavior changes.

## Design Consultant Subagent

Use a subagent as a design consultant when subagents are available and the task needs worldbuilding, flavor text, or player-facing narrative decisions.

Spawn a consultant for:
- shopkeeper dialogue or item flavor copy
- battle feature presentation text to players
- thematic fit checks ("does this feel like official Pokemon?")

Do not delegate code edits to this consultant. Keep all technical implementation in the main agent.

Use a prompt equivalent to:
- "Act as a Pokemon game design consultant. Do not write code. Propose dialogue, naming, and player-facing presentation for this feature. Keep output concise and setting-consistent."

Treat consultant output as design input, then implement technically in this repository.
